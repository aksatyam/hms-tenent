It handles walk-in registration, appointment check-in, digital queue management, clinical consultation with SOAP-based EMR, e-prescription with real-time drug interaction checking, lab order dispatch, and final discharge with automated billing. The Pharmacy module receives prescriptions in real-time via Kafka, manages drug dispensing with FIFO batch logic, enforces allergy and drug-interaction safety checks, maintains full batch traceability, automatically posts drug charges to Finance, and triggers procurement reorder workflows when stock falls below threshold.

📥 Prescription Receipt via Kafka
Kafka consumer listens on prescription.created topic. On event receipt, prescription decoded: patient UHID, drug list, dosage, duration. Pharmacist worklist updated via Socket.IO within <1 second. System queries drug master for each prescribed item. FIFO engine auto-selects oldest expiry batch first — batch number, expiry date, supplier, MRP displayed. Partial fulfillment workflow: dispense available, flag pending.

HR & Payroll Management

Biometric attendance integration, leave lifecycle management, automated payroll computation with PF/ESIC/PT deductions, and full Indian regulatory compliance reporting.

1
Employee Onboarding & Master Setup
HR executive creates employee record with Aadhaar + PAN (encrypted), medical council registration for clinical staff. System credentials provisioned automatically from role template. Biometric enrollment: fingerprint/face scan registered in ZKTeco/Suprema device. PF and ESIC number generated.
2
Shift Scheduling & Biometric Attendance
Branch Admin configures shift templates: morning, evening, night, rotational. Biometric API (ZKTeco/Suprema REST API) syncs punch records every 5 minutes: IN/OUT timestamps mapped to employee. Late, early-out, absent flags auto-raised. Manual override with justification requires Dept Head approval.
3
Leave Management Lifecycle
Employee applies for leave: CL/SL/EL/Comp-off/Maternity/Paternity. Approval workflow: Dept Head → Branch Admin override. Leave balance auto-deducted from accrual ledger. Sandwich leave rule applied. Leave encashment calculated and sent to payroll. Medical certificate upload required after 2 consecutive sick days.
4
Monthly Payroll Run & Computation
Engine aggregates: working days from attendance, approved leaves, OT hours, advances. Gross: Basic + HRA + TA + Special Allowances + OT. Deductions: PF (12%+12%), ESIC (0.75%+3.25%), Professional Tax (state slab), TDS, advance recovery. payroll.run Kafka event → Finance accounting entries.
5
Statutory Compliance Reports
Monthly: PF challan (ECR format for EPFO), ESIC Form 5. Quarterly: TDS Form 24Q. Annually: Form 16 per employee (Part A from TRACES, Part B auto-computed). Labour law registers maintained digitally. Professional Tax remittance challan per state. Full audit trail: all payroll changes with approver + timestamp.
M06
ADMIN

Inventory & Procurement

Multi-level PO approval, vendor management with rate contracts, GRN quality control, inter-branch transfers, and fixed asset lifecycle management.

1
Reorder Alert & Purchase Requisition
When stock ≤ reorder level (per item per branch), auto purchase requisition created. Emergency PRs (stock=0) escalated immediately to Branch Admin with SMS alert. Multiple items bundled into single PR. Last 3 purchase rates displayed for cost benchmarking.
2
Multi-Level Approval Workflow
Approval matrix configurable by tenant: PR up to ₹10k → Store Manager, ₹10k–₹50k → Branch Admin, ₹50k+ → Tenant Admin. Auto-escalation after 24h of inactivity. Emergency procurement: Branch Admin can bypass with mandatory post-approval within 24h.
3
Vendor Management & PO Generation
Vendor master with GST, PAN, bank details, performance rating, rate contracts. Rate contract items: PO price auto-populated. New item purchase: 3-quote comparison mandatory above threshold. PO dispatched to vendor via email API (Sendgrid/SES) + WhatsApp.
4
GRN — Goods Receipt & Quality Check
GRN created against PO. Item-by-item verification: quantity, batch numbers, expiry (rejection if <6 months for pharma), physical condition. Three-way match for payment: PO + GRN + vendor invoice. Partial GRN supported with follow-up tracking.
5
Asset Register & Consumption Tracking
Fixed asset register: serial number, purchase date, cost, department, warranty. Depreciation schedule auto-computed (SLM/WDV). AMC alerts 30/60 days before expiry. Breakdown record linked to asset: downtime tracked, vendor escalation auto-raised if SLA breached.

Business Intelligence & Reports

Real-time Kafka event ingestion, nightly ETL aggregation, executive dashboards, scheduled report delivery, and Power BI / Tableau export APIs.

1
Real-Time Event Stream Ingestion
All module Kafka events consumed by Analytics microservice → normalised event schema → written to Elasticsearch event index. Events carry: tenantId, branchId, moduleId, eventType, userId, entityId, timestamp. Redis cache serves real-time dashboard widgets with <200ms latency.
2
Nightly ETL Aggregation
Triggered by day-close.done Kafka event. Spring Batch job aggregates events into summary tables per branch and tenant: daily revenue, OPD volume, bed occupancy, pharmacy turnover, lab TAT compliance, staff attendance. Redis cache refreshed → dashboard load time <100ms.
3
Executive Dashboard — Real-Time View
Tenant Admin: cross-branch revenue comparison, OPD volumes, bed occupancy heatmap, doctor performance, outstanding bills. Branch Admin: daily collections, department OPD, staff attendance. All charts interactive — drill-down from tenant → branch → department → individual.
4
Scheduled Report Delivery
Report scheduler: daily, weekly, monthly, quarterly. Tenant Admin configures: report type, recipients, format (PDF/Excel/CSV), delivery time. Cron job generates reports → S3 with presigned URL → email via SES. Failed delivery retried 3 times with alerting.
5
BI Tool Export & Webhook Streaming
REST API for Power BI, Tableau, Looker Studio. API returns RBAC-filtered JSON — tenantId scope enforced at API layer. Webhook streaming: tenant registers webhook URL to receive real-time event stream. Rate-limited to prevent abuse. API key management with expiry and rotation workflows.

Audit, Compliance & Security

Immutable audit log for every data operation, DISHA data privacy lifecycle, AES-256 encrypted backups, OWASP hardening, and automated vulnerability scanning.

1
Immutable Audit Log Engine
Every CREATE, UPDATE, DELETE appended to audit_log: userId, tenantId, entity_type, entity_id, old_value (JSON), new_value (JSON), IP, user_agent, timestamp. Table is append-only — no UPDATE or DELETE permitted even for Super Admin (PostgreSQL row security). 7-year retention policy.
2
Security Monitoring & MFA
Failed login: 5 consecutive → account lockout + Admin alert. New device/IP login → OTP re-verification required. MFA mandatory for L1–L3. Concurrent session detection, single active session for sensitive roles. Security dashboard: real-time failed logins map, active sessions count, anomaly score per user.
3
DISHA Data Privacy Compliance
Patient consent lifecycle with version + timestamp. Right-to-access: FHIR bundle generated within 7 days. Right-to-correction: clinical staff reviews. Data retention: active patient data indefinitely, discharged data anonymised after 7 years. DPO (Data Protection Officer) access and reporting dashboard.
4
Backup & Disaster Recovery
Daily per-tenant PostgreSQL backup: pg_dump → AES-256 encrypted → S3 cross-region (Mumbai + Singapore). PITR: WAL archiving — restore to any second within last 7 days. RTO: 4 hours. RPO: 15 minutes. Monthly DR drill. Redis snapshot every 15 minutes. Elasticsearch snapshot daily.
5
Vulnerability Management & Pen Testing
OWASP ZAP: automated weekly scans — OWASP Top 10 checks. Snyk CVE: continuous container and dependency scanning. Annual 3rd-party penetration test by CERT-In empanelled agency. 72-hour breach disclosure workflow. Patch management SLA: Critical 24h, High 7d, Medium 30d.
